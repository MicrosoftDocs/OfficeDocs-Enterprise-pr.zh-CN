# <a name="deployment-advisors-for-office-365-services"></a>Office 365 服务的部署顾问

Office 365 部署顾问设置为您的组织的 Office 365 提供自我指导性的支持。只需选择一个向导，然后登录到 Office 365。 

当您选择的功能和要部署的选项，向导将建立到您的需要自定义的分步设置计划。您将获取一整套说明、 视频、 参考文章和脚本。某些向导具有时向导的其他区域中更改您的设置和数据使用向导可指导您将更改某些设置的自动化。即使在规划期间或之后已设置服务，以了解有关 Office 365 功能和选项的详细信息，可以随时随地都可以使用这些向导。
  
## <a name="windows-10-with-office-365"></a>使用 Office 365 的 Windows 10

[Microsoft 365 部署顾问](https://aka.ms/microsoft365setupguide)
  
Microsoft 365 是一个完整、 智能解决方案，包括 Office 365，Windows 10 和企业移动 + 安全。Microsoft 365 使所有人都是创作和在一起，安全地。使用本指南可设置 Windows 10 设备或与 Office 应用程序、 Windows 分析和 Windows Defender 高级威胁保护 （仅适用于 Microsoft 365 Enterprise E5 计划） 的可选部署升级到 Windows 10 的用户的计算机。

[Office 365 ProPlus 的部署顾问](https://aka.ms/proplusguidance)可帮助您获取有关最新版本的 Office 用户启动并正在运行。此顾问指导您完成为帮助用户直接从联机，Office 365 门户安装 Office 或到您的用户从本地位置部署 Office 的步骤。包含有关使用 System Center Configuration Manager (SCCM)，组策略启动脚本、 操作系统磁盘映像，说明或远程桌面服务 (RDS) 共享激活。

## <a name="mail-migration-and-protection"></a>邮件迁移和保护功能

### <a name="exchange-migration-advisor"></a>Exchange 迁移顾问
[Office 365 邮件迁移顾问](http://aka.ms/office365setup)将帮助您将邮箱从当前邮件系统移动到 Office 365 中 Exchange Online，使用自动化的工具和分步指南。建议将根据您当前的邮件系统，数邮箱想要迁移，以及如何制定计划来管理用户和用户访问您组织的最佳迁移路径。
  
[Gmail 联系人和日历助理](https://aka.ms/gmailcontactscalendar)提供有关迁移到 Office 365 的 Gmail 联系人和 Google 日历项目的分步说明。当用户的 Gmail 邮箱迁移到 Office 365 时，迁移电子邮件，但联系人和日历项目不是。此向导将 Gmail 联系人和 Google 日历项导入到 Office 365 提供的步骤。
  
### <a name="exchange-online-protection"></a>Exchange Online Protection
Microsoft [Exchange Online Protection (EOP)](https://docs.microsoft.com/Office365/SecurityCompliance/eop/exchange-online-protection-overview)是基于云的电子邮件筛选服务，可帮助保护您的组织免受垃圾邮件和恶意软件，并且包含功能来保护您的组织从消息策略违反行为。
  
## <a name="security-and-identity"></a>安全和标识

### <a name="azure-active-directory-connect-aad-connect-advisor"></a>Azure Active Directory 连接 （AAD 连接） 顾问
[Azure AD 连接顾问](https://aka.ms/aadconnectpwsync)是演示如何将用户帐户信息添加到 Office 365，而无需手动创建每个用户的分步指南。它还会显示如何配置密码哈希同步或传递身份验证，以便用户可以登录到其电子邮件和域使用相同的密码。如果您选择使用 AD FS 联合登录设置，本指南还提供步骤部署新服务器上或现有的 Windows Server 2012 R2 场中的 AD FS。AAD 连接替换较旧版本的标识集成工具，如目录同步和 Azure AD 同步，并主要用于 Office 365 添加到 Azure Active Directory 用户和其他数据。
  
### <a name="azure"></a>Azure
[Azure AD 基本安装指南](https://aka.ms/azureadbasic)指南可以帮助您设置了基于组的访问管理，自助服务密码重置的云应用程序和 Azure Active Directory 应用程序代理发布在本地 web 应用程序的功能。
  
[Azure AD Premium 安装指南](https://aka.ms/aadpguidance)可帮助您启用的 Azure Active Directory Premium （Azure AD Premium 或 AADP），它提供标识管理功能，例如多因素身份验证、 单一登录 (SSO) 设备注册的功能自助服务密码和组管理、 安全监控设备以及的详细信息。
  
### <a name="verify-your-domain"></a>验证您的域
[验证您的域中 Office 365](https://aka.ms/verifyyourdomaino365)顾问可帮助您自定义 Office 365 添加您自己的域名 （也称为您的网站名称）。
