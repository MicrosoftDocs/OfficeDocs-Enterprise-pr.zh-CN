## <a name="what-does-end-of-support-mean"></a>支持平均值内容的末尾？

Microsoft 产品，例如 PerformancePoint Server 2007 SP3、 ProClarity 软件和 SharePoint Server 2007 SP3 所有具有支持生命周期期间 Microsoft 提供的新功能、 修补程序、 和安全更新。产品生命周期通常会持续 10 年中的产品的初始发布日期，该生命周期的结束称为产品的结束的支持。ProClarity、 PerformancePoint Server 和 SharePoint Server 2007 已达到其结束的支持，将不再提供 Microsoft:
  
- 可能出现的问题的技术支持
    
- Bug 修复的问题的发现以及可能影响的稳定性和服务器的可用性
    
- 安全漏洞的发现和可能发出服务器或应用程序安全轻易地受到修补程序
    
- 时区更新
    
ProClarity、 SharePoint Server 2007 SP3 和 PerformancePoint Server 2007 SP3 的安装将继续运行，即使已结束支持。但是，我们强烈建议您迁移这些应用程序从尽快。

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>使用 Power BI （在内部部署或云）

Power BI 是一套业务分析工具以分析数据和分享见解。使用 Power BI，您可以创建交互式报表和仪表板使用本地或联机数据源。用户可以查看和使用报告和仪表板使用其计算机或移动设备。
  
Power BI 不包括在 Office 365 或 SharePoint Server，但单独的产品的包括 Power BI 桌面、 Power BI 网关和 Power BI 服务。Power BI 还将与 SharePoint Online 集成。您可以开始使用 Power BI 免费，并根据您的数据使用情况和业务需求，升级到 Power BI 专业人员。若要了解详细信息，请参阅[什么是 Power BI？](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>使用 Reporting Services （本地）

SQL Server Reporting Services 提供了一个稳固的报告解决方案，以及安装和配置本机模式或 SharePoint 集成模式下的 Reporting Services 的功能。您可以编写使用几种工具，包括报表设计器、 报表生成器和 Power View 的报告。SQL Server 最新版本中，您还可以使用 SQL Server 移动报告发布程序提供扩展的报告为任何屏幕大小，使您的组织能够使用其移动设备上的报告。若要了解详细信息，请参阅[SQL Server Reporting Services (SSRS): 创建、 部署和管理移动电话和分页报告](https://go.microsoft.com/fwlink/?linkid=841342)。
  
### <a name="use-performancepoint-services-on-premises"></a>使用 PerformancePoint Services （本地）

您知道，PerformancePoint Server 2007 已从 SharePoint Server 2007 单独购买。PerformancePoint Services 与 SharePoint Server 2010 从开始，是在 SharePoint Server 中的服务应用程序。这意味着不需要购买单独的服务器许可证或硬件，才能使用 PerformancePoint Services。
  
要从 PerformancePoint Server 2007 移动到 PerformancePoint Services，您将移动到 SharePoint Server 的较新版本和配置 PerformancePoint Services。移动到的 SharePoint Server 的版本将确定是否可以导入现有仪表板内容从 PerformancePoint Server 2007 到 PerformancePoint Services。
  
- 如果您正在升级到 SharePoint Server 2010 中，您可以从导入 PerformancePoint 仪表板内容 PerformancePoint Server 2007 到 SharePoint Server 2010 中的 PerformancePoint Services。若要了解有关此工作原理的详细信息，请参阅[导入向导： PerformancePoint Server 2007 内容到 SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873)。
    
- 如果您正在移动到 SharePoint Server 2013 或 SharePoint Server 2016，很可能需要创建新仪表板内容 （数据源、 报表、 记分卡和仪表板页面）。
    
若要开始在您的 PerformancePoint Services 升级计划，请参阅以下资源：

2. 如果您知道哪个版本的 SharePoint 您移动以，请参阅为 PerformancePoint Services 的相应文章：
